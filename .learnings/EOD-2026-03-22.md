## Key Findings

### [FND-20260322-001] HKJC Session Rate-Limiting (Critical)

**Finding**: HKJC blocks rapid sequential navigation within the same Chrome session.

**Details**:
- Fresh Chrome (brand new session) → **~2s** for 10 races
- Reused session within continuous mode → **~15s** (HKJC throttles)
- After ~5-10 minutes continuous scraping, session becomes unreliable
- Adding wait time (e.g., 3s warmup) helps but doesn't fully solve continuous reuse

**Implication**: Session-reuse pattern is fundamentally limited by HKJC's rate-limiting. Fresh browser per cycle is more reliable despite being ~5s vs ~2s.

**Status**: resolved — adopted fresh browser per cycle

---

### [FND-20260322-002] Socket.IO 4.x Namespace Bug

**Finding**: `io('https://host/socket.io')` makes Engine.IO requests to `/socket.io/socket.io/` (double path).

**Root Cause**: When the URL ends with `/socket.io`, Socket.IO client appends `socket.io` again as the default Engine.IO path.

**Fix**: Use `io(origin, { path: '/socket.io/' })` — separate origin from path.

**Status**: resolved

---

### [FND-20260322-003] JS Object Key Serialization — Always Strings

**Finding**: JavaScript `Object.keys()` and `JSON.stringify()` always convert numeric object keys to strings. `{1: 'a', 10: 'b'}` becomes `{"1":"a","10":"b"}`.

**Impact**: 
- Scraper returns `"01", "02"` string keys
- MongoDB stores as strings
- racecard_entries uses numeric `horse_no: 1`
- Frontend lookup `oddsData[horseNo]` with numeric key fails for string keys

**Fix**: Normalize at every boundary — scraper broadcasts `Number()`, server uses `Number()`, frontend uses `String()` consistently.

**Status**: resolved

---

### [FND-20260322-004] Sparkline History Format Mismatch

**Finding**: `HorseSparkline` component expected `[{time, win, place}]` array format but MongoDB returns `{win: [], place: []}` parallel arrays.

**Impact**: Render crashed silently → blank chart panel on page load.

**Fix**: Transform in component: `raw.win.map((w,i) => ({win: w, place: raw.place[i]}))`

**Status**: resolved

---

### [FND-20260322-005] Server Double-listen() Crash

**Finding**: `index.cjs` had two `httpServer.listen()` calls — one at the old `app.listen()` location (line 279) and one in the new `connect().then()` chain. Second listen() threw `EADDRINUSE`.

**Impact**: Server crashed on every start. launchd reported exit code 1 repeatedly.

**Status**: resolved

---

### [FND-20260322-006] MongoDB connect Race Condition

**Finding**: `connect().catch(console.error)` was non-blocking, but `httpServer.listen()` was called synchronously after. Server accepted connections before MongoDB was ready.

**Impact**: All DB queries failed immediately on startup.

**Fix**: Chain `connect().then(() => httpServer.listen(...))`.

**Status**: resolved

### [FEAT-20260322-002] odds_collector: oddsCache TTL Cleanup

**Request**: `oddsCache` in server grows indefinitely. Should evict old race caches.

**Complexity**: medium

**Suggested**: Add TTL (e.g., 1 hour after race ends) or max-races limit with LRU eviction.

### [FEAT-20260322-003] MongoDB Live Odds: Deduplication

**Request**: Scraping the same race into MongoDB with no deduplication creates many redundant docs. Could store only latest odds per race+horse combination.

**Complexity**: medium

**Suggested**: Upsert instead of insertMany, keyed by `race_id + horse_no + scraped_at` minute bucket.

**Complexity**: simple

## Issues Found During FEAT-009 Mobile UI Testing (2026-03-23)

### Race Switching Broken (FEAT-010 Regression)
- **Symptom**: Switching races (R1→R2) doesn't clear old predictions/racecard
- **Root**: No cleanup effect on raceId change for predictions/racecardData state
- **Fix**: Added `useEffect(() => { setPredictions([]); setRacecardData(null); }, [raceId])` in App.jsx
- **Lesson**: Every stateful data (predictions, oddsData, oddsHistory) needs cleanup on raceId change

### Mobile Table Column Alignment
- **Symptom**: Data showing in wrong columns when switching races (stale state)
- **Root**: Same as above — stale predictions during race transition
- **Fix**: Included in raceId cleanup effect

### CSS Class Conflicts
- **Issue**: `.rank-N` classes conflicting between desktop/desktop-only and mobile
- **Fix**: Separate class names for mobile (`ut-*`) vs desktop (`predicted-rank`, `rank-*`)
- **Lesson**: Mobile/desktop components need distinct class prefixes
